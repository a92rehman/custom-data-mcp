Self-Healing Detail
What happens step by step when something breaks
Two parallel recovery systems — one for query errors, one for infrastructure.

🔧 Loop 1: Query Self-Healing

A governed SQL query fails at BigQuery

1
execute_query returns error JSON — includes error_class, message, table/column referenced, event_id
2
data-analyst opens a ticket — report_ticket(loop="query", category=schema/syntax/partition/cost)
3
Dispatches query-fixer with: original question, failed SQL, error JSON, table schema, rule file path
4
query-fixer reads actual schema + rules — diagnoses the problem (renamed column? missing filter? syntax typo?)
5
Proposes corrected SQL v2 — must be different from v1. Parent re-executes via MCP.
6
Success? Ticket closed as auto_fixed. Still fails? Retry up to 3 total. After 3: escalate.

⚡ Loop 2: System Self-Healing

Infrastructure issue detected by hook or during query

1
Session hook runs update.py — syncs rules from latest git tag, then runs silent auto-heal
2
Auto-heal fixes what it can — missing env (recovers from audit log), broken paths, stale plugin.json, stackdumps
3
system-doctor activates — auto-dispatched by data-analyst on system-level errors or after 3 failed query fixes
4
For each symptom: DETECT → DIAGNOSE → HEAL — applies auto-fix, logs action to ticket
5
VERIFY — re-runs the failing check. If passed: ticket = auto_fixed.
6
After 2 failed fixes: ESCALATE — auto-files GitHub issue with sanitized evidence (rate limited: 1/symptom/day)
Agents
4 agents, each with a clear job
They work together but never overlap. Each has specific tools and triggers.

📊 data-analyst

Primary — all data questions
Triggers
  • Any question about Taleemabad data
  • "How many teachers?", "LP rate?", "FICO scores?"
What it does
  • Phase 1: Reads governance rules for the right region/domain
  • Phase 2: Asks mandatory clarification questions
  • Phase 3: Generates governed SQL (never ad-hoc)
  • Phase 4: On error, opens ticket + dispatches query-fixer
ReadGlobGrep

🛠 data-admin

Detective — infrastructure diagnostics
Triggers
  • "What tables are in RUMI_DB?"
  • "When was coaching_observation updated?"
  • "Show me audit logs" / "Query costs?"
What it does
  • Browses schemas, checks table freshness
  • Queries audit logs for cost and error patterns
  • Compares actual schema to rule definitions
All MCP tools

🔧 query-fixer

Mechanic — fixes broken SQL
Triggers
  • Auto-dispatched by data-analyst Phase 4
  • Only for query-level errors (not system issues)
  • Never called directly by users
What it does
  • SCHEMA_DRIFT: finds renamed columns from actual schema
  • MISSING_PARTITION: adds date filter from rule file
  • SYNTAX_ERROR: fixes typos, functions, backticks
  • COST_EXCEEDED: narrows date range or switches table
  • Gives up after 3 attempts (never returns same SQL)
ReadGlobGrep

⚡ system-doctor

Healer — fixes infrastructure
Triggers
  • Auto-dispatched by data-analyst on system-level errors
  • Auto-dispatched after 3 failed query fix attempts
  • Manual fallback: /taleemabad-doctor command
What it does
  • Checks MCP connectivity, env config, rules sync
  • Fixes: re-run hook, setx env var, delete stackdump
  • Verifies fixes worked, escalates if not
  • Files GitHub issues (rate limited, sanitized)

Error Classification
8 error classes drive the fix strategy
Query errors go to query-fixer. System errors go to system-doctor. Each opens a ticket.
SCHEMA_DRIFT
Column renamed/removed, table not found
→ query-fixer reads schema, finds replacement
MISSING_PARTITION
Query on partitioned table without date filter
→ query-fixer adds filter from rule file
SYNTAX_ERROR
SQL typo, wrong function, mismatched parens
→ query-fixer corrects syntax
COST_EXCEEDED
Query scans more bytes than limit
→ query-fixer narrows range or switches table
PERMISSION_DENIED
No access to table or dataset
→ system-doctor (not a query problem)
TIMEOUT
Query deadline exceeded
→ system-doctor (infrastructure issue)
BIGQUERY_UNAVAILABLE
503 / service down
→ system-doctor (service outage)
OTHER
Unrecognized exception
→ Manual diagnosis needed
