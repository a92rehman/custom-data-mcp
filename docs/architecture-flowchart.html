The system-doctor agent handles 9 infrastructure symptoms with the DETECT → DIAGNOSE → HEAL → VERIFY cycle
Symptom Detection Auto-Fix Fallback
connection_failed curl /health returns non-200 Retry after 10s Tell user to check internet
user_env_missing ~/.claude/taleemabad-data-mcp.env not found Recover email from audit log Ask user for email → /taleemabad-setup
user_env_unexpanded Header contains literal ${TALEEMABAD_USER} setx (Win) / shell rc (Linux) Tell user to restart terminal
rules_path_missing ~/.claude/taleemabad-rules-path gone or stale Re-run update.py hook Suggest reinstall after 2 failures
rules_stale Local version 2+ behind latest git tag Force refresh via hook —
plugin_not_installed claude plugin list missing taleemabad-data — Print install commands
plugin_outdated Local plugin 2+ minor behind — Offer claude plugin update
mcp_handshake_fail Server up but /mcp disconnected Retry once Escalate to GitHub
hook_crashed bash.exe.stackdump found or hook log errors Switch to Python hook —
